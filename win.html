Planos GitHub

Sobre os Planos do GitHub: GitHub oferece planos gratuitos e pagos para armazenar e colaborar no código. Alguns planos estão disponíveis apenas para contas pessoais, enquanto outros planos estão disponíveis apenas para as contas de organização e corporativas.

GitHub Free: Com o GitHub Free para contas pessoais, você pode trabalhar com colaboradores ilimitados em repositórios públicos ilimitados com um conjunto completo de recursos e em repositórios privados ilimitados com um conjunto de recursos limitado.
Com o GitHub Free, sua conta pessoal inclui:

  • GitHub Community Support
  • Dependabot alerts
  • Regras de proteção contra implantação para repositórios públicos
  • Imposição da autenticação de dois fatores
  • Armazenamento de 500 MB do GitHub Packages
  • 120 horas de núcleo dos GitHub Codespaces por mês
  • 15 GB de armazenamento dos GitHub Codespaces por mês
  • Recursos do GitHub Actions:

GitHub Pro: Além dos recursos disponíveis no GitHub Free para contas pessoais, o GitHub Pro inclui:

  • Suporte do GitHub via e-mail
  • 3 mil minutos de GitHub Actions por mês
  • Armazenamento de 2 GB do GitHub Packages
  • 180 horas de núcleo dos GitHub Codespaces por mês
  • 20 GB de armazenamento dos GitHub Codespaces por mês
  • Ferramentas avançadas e insights em repositórios privados

GitHub free para organizações: Com o GitHub Free para organizações, você pode trabalhar com colaboradores ilimitados em repositórios públicos ilimitados com um conjunto completo de recursos ou em repositórios privados ilimitados com um conjunto de recursos limitado.
Além dos recursos disponíveis no GitHub Free para contas pessoais, o GitHub Free para organizações inclui:

  • GitHub Community Support
  • Controles de acesso da equipe para gerenciamento de grupos
  • 2 mil minutos de GitHub Actions por mês
  • 500 GB de armazenamento do GitHub Packages

GitHub team: Além dos recursos disponíveis no GitHub Free para organizações, o GitHub Team inclui

  • Suporte do GitHub via e-mail
  • 3 mil minutos de GitHub Actions por mês
  • 2 GB de armazenamento do GitHub Packages
  • Ferramentas avançadas e insights em repositórios privados
  • A opção de habilitar ou desabilitar GitHub Codespaces

GitHub Enterprise: O GitHub Enterprise inclui duas opções de implantação: o GitHub Enterprise Cloud, que é hospedado na nuvem e faz parte do GitHub.com e o GitHub Enterprise Server, que é auto-hospedado.
Além dos recursos disponíveis no GitHub Team, o GitHub Enterprise inclui:

  • Suporte do GitHub Enterprise
  • Segurança adicional, conformidade e controles de instalação
  • Autenticação com o logon único do SAML
  • Provisionamento de acesso com SAML ou SCIM
  • Regras de proteção contra implantação com o GitHub Actions para repositórios privados ou internos
  • GitHub Connect
  • A opção de comprar GitHub Advanced Security.

O GitHub Enterprise Cloud inclui especificamente:

  • 50 mil minutos de GitHub Actions por mês
  • 50 GB de armazenamento do GitHub Packages
  • Um contrato de nível de serviço de tempo de atividade mensal de 99,9%
  • A opção de gerenciar de forma centralizada a política e cobrança para várias organizações GitHub.com com uma conta corporativa
  • A opção de provisionar e gerenciar as contas de usuário para seus desenvolvedores por meio do Enterprise Managed Users

Benefícios do GitHub

O GitHub oferece uma variedade de benefícios e recursos que o tornam uma ferramenta poderosa para desenvolvedores e equipes de desenvolvimento. Aqui estão alguns dos principais benefícios e recursos do GitHub:

Controle de Versão com Git: O GitHub utiliza o sistema de controle de versão distribuído Git, que permite rastrear alterações no código-fonte, facilitando a colaboração entre desenvolvedores.

Colaboração: O GitHub facilita a colaboração em equipe. Várias pessoas podem trabalhar no mesmo projeto simultaneamente, cada uma em seu próprio branch, e depois mesclar suas alterações usando solicitações de pull.

Solicitações de Pull (Pull Requests): Essa funcionalidade permite que os desenvolvedores proponham mudanças em um branch e solicitem que essas mudanças sejam mescladas no branch principal. Isso é essencial para a revisão de código e a integração de contribuições.

Issues e Sistema de Rastreamento: As issues são usadas para rastrear bugs, tarefas e melhorias no projeto. Isso facilita a comunicação entre os membros da equipe e a organização do trabalho a ser feito.

Integração Contínua: O GitHub Actions permite configurar e automatizar fluxos de trabalho de integração contínua, teste e implantação diretamente no GitHub.

Proteção de Branch: É possível configurar regras de proteção em branches específicos, garantindo que determinadas condições (como revisão de código ou aprovação de solicitação de pull) sejam atendidas antes que as alterações possam ser mescladas.

GitHub Pages: Oferece a capacidade de hospedar sites estáticos diretamente a partir do repositório, permitindo que você publique documentação, páginas de projetos, ou até mesmo um site para o seu projeto.

Gists: Permite compartilhar trechos de código ou notas rapidamente, sem a necessidade de criar um repositório completo.

Comunidade: O GitHub é uma grande comunidade que hospeda milhões de projetos de código aberto. Isso facilita a descoberta de projetos interessantes, colaboração com outros desenvolvedores e aprendizado através do código de terceiros.

Segurança: O GitHub oferece ferramentas para análise de segurança do código, alertas de vulnerabilidades e integrações com ferramentas de varredura de segurança.

Gráficos de Contribuição: Fornece gráficos visuais que mostram a atividade de contribuição ao longo do tempo, destacando os principais contribuidores e as tendências de desenvolvimento.

Integração com Outras Ferramentas: O GitHub pode ser integrado a uma variedade de ferramentas e serviços, como Slack, Trello, JIRA, entre outros, para facilitar a comunicação e o gerenciamento de projetos.

Esses são apenas alguns dos muitos benefícios e recursos que o GitHub oferece. A plataforma continua evoluindo com novos recursos e melhorias para atender às necessidades em constante mudança dos desenvolvedores e equipes de desenvolvimento.
